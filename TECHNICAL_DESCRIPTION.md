# Техническое описание проекта Вкусная Доставка

## 1. Общая информация

- **Название проекта:** Вкусная Доставка (food-delivery-bot)
- **Цель проекта:** VK-бот для ресторана/службы доставки еды в городе Родники Ивановской области. Бот принимает заказы от клиентов, CRM-панель для управления заказами, уведомления для персонала (кухня, курьеры).
- **Целевая аудитория:** Клиенты ресторана через VK Messenger; администратор, кухня, курьеры — через VK личные сообщения.

## 2. Технический стек

- **Язык программирования:** Python 3.14 (PC) / 3.13 (Termux Android)
- **Фреймворки/библиотеки:**
  - vkbottle >=4.3, <5.0 — VK бот-фреймворк
  - FastAPI 0.109.0 — CRM API
  - uvicorn 0.27.0 — ASGI-сервер
  - SQLAlchemy >=2.0.36, <2.1 — ORM (async)
  - aiosqlite 0.19.0 — async SQLite-драйвер
  - httpx 0.26.0 — HTTP-клиент (для YandexGPT API)
  - pydantic >=2.0, <3.0 — валидация моделей
  - python-dotenv 1.0.0 — загрузка .env
  - pytest >=7.0, pytest-asyncio >=0.21 — тесты
- **База данных:** SQLite (async, файл `food_delivery.db`)
- **Хостинг/сервер:** Android-телефон (Termux) + FastAPI на порту 8080. Доступ с PC по Wi-Fi (192.168.31.36:8080).
- **API и интеграции:**
  - VK Bot API (Long Polling) — приём/отправка сообщений
  - YandexGPT API (`llm.api.cloud.yandex.net`) — ИИ-консультант по меню
  - VK Callback API (`/callback`) — альтернатива Long Polling (доступна, но не используется)

### Поток данных

1. **Клиент** → VK Messenger → **Бот** (Long Polling) → обработка сообщения
2. Бот → меню, корзина, оформление заказа → **SQLite DB**
3. Заказ → уведомление **Админу** (VK DM) → Принятие → **Кухне** → Готов → **Курьеру** → Доставлен
4. **CRM** (React SPA на порту 8080) → FastAPI → SQLite → просмотр/управление заказами
5. **ИИ** → YandexGPT → консультация по меню (не управляет флоу заказа)

## 4. Функционал

### Команды бота

| Команда | Описание |
|---------|----------|
| `/start`, приветствие | Главное меню |
| 🍔 Меню | Показать категории меню |
| 🛒 Корзина | Просмотр корзины с кнопками +/- |
| 📦 Мои заказы | История заказов пользователя |
| ❓ Помощь | Справка |
| Текст (название блюда) | Добавить в корзину (фаззий-матчинг) |
| `Пепперони 4, Кола 2` | Добавить несколько позиций за раз |
| `- пицца` | Убрать пиццу из корзины (на 1) |
| ➕/➖ кнопки | Изменить количество в корзине |
| ✅ Оформить заказ | Начать оформление |
| 🚗 Доставка / 🚶 Самовывоз | Выбор типа получения |
| Адрес доставки | Текстом (город/ближняя зона) |
| 💳 Карта / 💵 Наличные | Способ оплаты |
| 🔔 Все статусы / ⚡ Только ключевые | Настройка уведомлений |

### Сценарии обработаны

- Просмотр меню по категориям (Пицца, Рамен, Салаты, Бургеры, Снэки, Напитки)
- Добавление товаров в корзину текстом (фаззий-матчинг по названию)
- AI-парсинг: "Хочу пиццу и колу" → добавление в корзину через YandexGPT
- Управление корзиной: +/- кнопки, текстовые команды, очистка
- Оформление заказа: тип получения → адрес → телефон → оплата → подтверждение
- Зоны доставки: Город Родники (200₽, бесплатно от 1000₽), ближняя зона (300₽)
- Уведомления клиенту по статусам (все / только ключевые)
- Уведомления персоналу: Админ → Кухня → Курьер (через VK DM)
- CRM панель: заказы, меню (CRUD), статистика, сотрудники, логи бота
- Авторизация CRM по API-ключу

### Что ещё нужно доделать

- Termux:Boot автозапуск (скрипт готов, не настроен на телефоне)
- Интеграция с терминалами/эквайрингом (пока только наличные/карта при получении)
- Уведомление курьеру с адресом и временем доставки (базовая реализация есть)
- Тесты на интеграцию с VK API (пока unit-тесты)

## 5. ИИ-часть

- **Модель:** YandexGPT Lite (`yandexgpt-lite`) через Foundation Models API
- **Endpoint:** `https://llm.api.cloud.yandex.net/foundationModels/v1/completion`
- **Авторизация:** Api-Key header (YANDEX_GPT_API_KEY + YANDEX_GPT_FOLDER_ID из .env)
- **Как обрабатываются запросы:**
  1. Пользователь пишет сообщение
  2. Если не распознано как команда → `handle_ai_chat()`
  3. История (последние 6 сообщений) отправляется в YandexGPT
  4. Ответ очищается от JSON-фрагментов и истории диалога
  5. Если AI распознал заказ (JSON с items) → добавляется в корзину
  6. Иначе → показывается текстовый ответ
- **Промпт:**
```
Ты - помощник ресторана "Вкусная Доставка".
- Консультируй по меню, помогай с выбором
- НЕ управляй флоу заказа (адрес, оплата, телефон — бот сам)
- НЕ выводи JSON, отвечай только текстом
- Краткие ответы, 2-3 предложения
```
- **Ограничения:** AI НЕ управляет оформлением заказа, НЕ спрашивает адрес/оплату, НЕ подтверждает заказ. Это делает бот.

## 8. Для продажи

### Уникальные фичи
- **Автономный бот на телефоне** — работает 24/7 без VPS, на старом Android
- **CRM из коробки** — встроенная веб-панель с авторизацией, не нужен отдельный хостинг
- **AI-консультант** — помогает клиентам выбирать блюда через YandexGPT
- **Уведомления персоналу** — автоматическая цепочка: админ → кухня → курьер
- **Зоны доставки** — настраиваемые зоны с автоматическим расчётом стоимости
- **Мгновенный деплой** — `git clone` + `pip install` + `nano .env` = 5 минут

### Преимущества перед конкурентами
- Не нужен VPS/сервер — работает на телефоне
- Встроенная CRM — не нужно покупать/настраивать отдельно
- AI-помощник — клиент получает помощь в выборе, а не просто кнопки
- Готов к запуску — можно начать принимать заказы прямо сейчас
- Легко модифицировать — чистый Python, понятная архитектура

### Готовность к запуску
**~85%** — бот работает, CRM работает, заказы принимаются. Не хватает:
- Терминальной оплаты (карта при получении работает)
- Termux:Boot (автозапуск)
- Нормального тестирования на реальных пользователях (2-3 дня)
